# Mission

## Target audience
Users of OpenCode CLI who want to build reliable, review-driven projects using Spec-Driven Development.

## Problem
"Vibecoding" — building software without structured review or validation — leads to hidden bugs, poor architecture, and difficult maintenance. Developers need a workflow that enforces human-in-the-loop review at every stage.

## Solution
`sdd-work` skill automates the SDD workflow in OpenCode:
1. Constitution — define mission, stack, roadmap
2. Feature spec — plan features with requirements and validation criteria
3. Implementation — build per task groups
4. Validation — automated AI code review (`@code_checker`) + human approval
5. Replanning — iterate on constitution and process

No step proceeds without explicit human approval.

## CV context
This repository demonstrates:
- **Spec-Driven Development** methodology in practice
- **Human-in-the-loop** AI-assisted workflow
- **Structured project management**: specs → implementation → validation → iteration
- **Tooling**: Python, git/GitHub, OpenCode, AI code review agents
- **Documentation-first approach**: every decision is recorded and versioned
